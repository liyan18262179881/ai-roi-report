# 决策日志

按时间顺序记录这个项目演进中做过的关键决定与背后的取舍,方便回溯"为什么是现在这样"。

## 从想法到 MVP

- **最初想法**:做一个 AI ROI 决策工具,帮中小电商在投钱做 AI 前算清"值不值、怎么做、花多少、多久回本"。
- **商业化路径**:工具 + 服务打包。先用工具评估建立信任 → 卖落地实施服务 → 沉淀数据与方法论 → 产品化/白标。对从零起步者,先拿到第一个付费客户和真实案例比覆盖面更重要。
- **MVP 范围之争**:曾纠结"1 个场景做透" vs "多场景做浅"。结论:先做透高频场景,其余做入口收集需求信号。后续扩展为完整多场景报告生成器。

## 产品形态演进

1. **v1**:交互式 PRD 文档(`legacy/prd-v1.html`)—— 用户觉得太重(10 章节)。
2. **v2**:简化为双视角计算器(老板/我的)。
3. **v3**:扩展为电商多团队计算器(视觉/客服/销售),系数与单价改用 2026 数据(`legacy/calculator-v3.html`)。
4. **当前**:升级为"商业计划报告生成器"(`index.html`)—— 真正像一份可卖的咨询交付物。

## 关键取舍记录

- **销售字段抽象 → 可理解**:早期让用户填"转化提升%"被吐槽看不懂。改为只填营收/毛利率,增收幅度用行业基准预填。
- **业务量从"按人数偷偷推算" → 用户直填**:用户质疑"没填出图量怎么算出价格的?"。改为直填业务量 + 单价摆明面,价格 = 业务量 × 单价,全程可追溯。
- **系数下放后台**:产能倍数、AI 处理率等用户填不出,移到后台按行业基准取值,报告里注明。
- **不编造**:用户要求没填的不要编。改 `val()` 空返回 null,报告标"待补充"且不计入合计。
- **补回"增效"口径**:重构时一度只剩"降本"。按早期决策补回视觉的"产能翻倍/白赚产出"叙事。
- **投入侧必须有**:用户指出"只算降本不算投入不是合格商业报告"。新增自建 vs 外包对比,自建团队按 2026 真实薪资、随方向数缩放。
- **设计角色按全职算**:曾按 0.5 人折算导致"月薪1万、小计5000"看不懂,改为全职 1 人。
- **导出 PDF 而非打印**:用 html2pdf 直接生成文件下载。
- **企查查/财务**:明确不自动拉取、不伪造真实公司财务,公司信息由用户填。

## 待决 / 可讨论

- 客服 AI 自动处理默认值取 45%(稳健)。面对保守客户可降到 30%,卖标杆方案可回到 60%。
- 自建团队默认配置面向中小企业应用岗;客户体量差异大时需调整薪资与人数。
